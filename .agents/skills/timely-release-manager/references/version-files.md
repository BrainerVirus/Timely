# Version Files Inventory

All five files that must stay in sync on every release. Update the three hand-edited files manually, then run the commands to regenerate the two lock files.

## Hand-edited files

### `package.json`

- `"version"` field — set to new version string (e.g., `"1.2.3-beta.1"`)
- `"dev:prerelease"` script — update embedded version if it contains a hardcoded value
- `"tauri:dev:prerelease"` script — same as above

### `src-tauri/Cargo.toml`

- `version` field under `[package]` — e.g., `version = "1.2.3-beta.1"`

### `src-tauri/tauri.conf.json`

- `"version"` field at the top level — e.g., `"version": "1.2.3-beta.1"`

## Auto-regenerated files (do not hand-edit)

### `package-lock.json`

Regenerated by:

```bash
npm install --package-lock-only
```

This updates the `"version"` field in `package-lock.json` to match `package.json` without touching `node_modules`.

### `src-tauri/Cargo.lock`

Regenerated by:

```bash
cd src-tauri && cargo check
```

This updates the `version` entry for the Timely package in `Cargo.lock` without a full rebuild.

## Version string format

| Release type  | Example string         |
|---------------|------------------------|
| Beta          | `1.2.3-beta.1`         |
| Release candidate | `1.2.3-rc.1`       |
| Stable        | `1.2.3`                |

The `v` prefix is used only for git tags and GitHub release titles (e.g., `v1.2.3-beta.1`).  
The highlights key, Cargo.toml, tauri.conf.json, and package.json all use the bare version without `v`.

## Verification

After bumping, do a quick sanity check:

```bash
# All four version strings should match
node -p "require('./package.json').version"
grep '^version' src-tauri/Cargo.toml
jq '.version' src-tauri/tauri.conf.json
```
